# Packages: LWPT-canonical workspace projects; consumed by any LWPT-using project (GocciaScript first)

Supersedes [ADR-0003](./0003-vendored-permanent-fork-graduation.md). The earlier model treated GocciaScript as an upstream that LWPT vendored from + submitted courtesy patches back to; the workspace packages under `packages/<name>/` were "permanently forked" copies of GocciaScript-canonical source. That framing was wrong from the start — **LWPT and GocciaScript are sister projects under the same owner, with LWPT's `packages/<name>/` already the canonical source for every shared utility** (HTTPClient, CLI, Semver, TOML, TestingPascalLibrary, TransportSecurity, FileUtils, StringBuffer, Platform, OrderedStringMap, BaseMap, `Shared.inc`). Some are byte-identical to GocciaScript's older copies (`TransportSecurity`, `FileUtils`, `StringBuffer`, `TestingPascalLibrary`, `OrderedStringMap`, `BaseMap`); five have meaningfully diverged with LWPT ahead (`HTTPClient` with the byte-safety fixes; `CLI.Options` + `CLI.Parser` with the prefix-strip + dead-code removal + space-separated value widening; `Semver` + `Platform` renamed from their `Goccia.*` form; `TOML` renamed + parser-class refactored). ADR-0017 re-anchors on reality: **a package is the umbrella term** (no separate "shared package" / "vendored package" / "graduated package" categories); **consumption is a property of who uses it**, not of the thing itself; **LWPT is the canonical source** for the packages it ships; **any LWPT-using project consumes the same way** via `lwpt install`; **GocciaScript is the first named adopter** and commits to Path A (full toolchain adoption — build, install, test, format). The reframe is documentation + a patch-marker sweep + a transition-policy commitment, with zero code-architecture change. The Hard Constraint in [`AGENTS.md`](../../AGENTS.md) becomes **"Packages own their contents"** + **"No patch markers"**; [`docs/vendored.md`](../packages.md) is renamed to [`docs/packages.md`](../packages.md) under the new framing; [`CONTEXT.md`](../../CONTEXT.md) retires the `Vendored` glossary entry; the `[gpm patch]` / `[LWPT patch]` markers in `packages/httpclient/` + `packages/cli/` + `packages/semver/` + `source/Platform.pas` are swept (rewritten as plain Pascal comments preserving the rationale).

## Considered Options

### Ownership model — what is a package, and where's the canonical source?

- **Vendored-from-upstream + permanent-fork model** (the ADR-0003 framing this ADR replaces). `packages/<name>/` is a frozen copy of GocciaScript source, patched locally, graduating to a standalone repo someday. Rejected on contact with reality: LWPT and GocciaScript are co-owned; there is no upstream to defer to; calling LWPT's already-improved versions "patches against upstream" inverts the actual direction of canonical-ness.
- **Packages as canonical-at-LWPT, consumed by any LWPT-using project.** *Chosen.* `packages/<name>/` is the canonical source. LWPT consumes them as workspace deps via `[workspaces]`; future consumers (GocciaScript via `lwpt install`, third parties via the same once Phase 2 lands standalone repos) consume them the same way. No "consumer tier" distinction — a package's identity doesn't depend on who consumes it.
- **Per-package "is this shared?" intent flag** (`[package] shared = true`). Rejected: invents a category to gate behaviour that doesn't actually differ. Every package is potentially shareable; how it's actually used is a runtime property of the consumer's manifest.

### Consumer relationship — how do other projects pick up the canonical packages?

- **GocciaScript adopts LWPT in full (Path A).** *Chosen.* `lwpt build / install / test / format` becomes GocciaScript's toolchain; `build.pas` + `format.pas` retire; GocciaScript's `lwpt.toml` declares `[build]` entries for `GocciaScriptLoader`, `GocciaTestRunner`, `GocciaREPL`; `lwpt install` symlinks LWPT-canonical packages into GocciaScript's `.lwpt/modules/<name>/`. The non-Pascal-test-runner phases (GocciaTestRunner runs `.js` acceptance tests; LWPT's `lwpt test` discovers Pascal `*.Test.pas` files only) coexist — GocciaTestRunner becomes a `[build]` entry plus a `[run-js-tests] script = "scripts/run-acceptance.pas"` callable via `lwpt run` (ADR-0013).
- **Path B: packages only, `build.pas` stays.** Rejected as the final state; survived as a possible interim if Path A's migration is staged.
- **No commitment to GocciaScript adoption; leave it to GocciaScript's owner to decide.** Rejected: the user explicitly committed to Path A. ADR-0017 documents the commitment so the cross-project work has a written reference.
- **Tier consumers (LWPT-internal / sister-project / third-party).** Rejected: consumer tiers re-introduce the "vendored = special category" mistake at the package-consumption layer.

### Transition — what happens to GocciaScript's older copies of the diverged packages?

- **Freeze GocciaScript's copies.** *Chosen.* All future improvements to `HTTPClient`, `CLI.*`, `Semver`, `Platform`, `TOML`, `TestingPascalLibrary`, `TransportSecurity`, `FileUtils`, `StringBuffer`, `OrderedStringMap`, `BaseMap`, or `Shared.inc` land in LWPT-canonical (`packages/<name>/source/`) only. GocciaScript's `source/shared/*` (and the two `source/units/Goccia.{Semver,Platform,TOML}.pas` files) are explicitly frozen pending Path A adoption. Backports happen only on demand for P0 bugs, with the LWPT-canonical version always landing first; the backport commit message names the LWPT-canonical commit and notes "drop on Path A adoption".
- **Continuous one-way sync LWPT → GocciaScript.** Rejected: every LWPT-canonical change forces an additional GocciaScript commit; obscures what's actually canonical; creates the illusion that GocciaScript's copies are kept-current-by-policy when they're functionally stale.
- **Bidirectional sync.** Rejected: recreates the drift problem this ADR fixes.
- **No transition policy.** Rejected: leaves the divergence undocumented and Path A timeline ambiguous.

### Patch markers — `{ [gpm patch] }` and `{ [LWPT patch] }`

- **Drop both markers entirely.** *Chosen.* With no upstream to mark deltas against, the marker syntax loses its semantics. Git history is the canonical record of every change (what + when + why). The explanatory text that historically accompanied each marker stays as a plain Pascal comment — only the `[gpm patch]` / `[LWPT patch]` prefix goes.
- **Keep both markers, reframe semantics** (`[gpm patch]` = "needs to land in GocciaScript when it adopts"; `[LWPT patch]` = "transformation at extraction time"). Rejected: distinction between bug-fix-to-port and rename-style-transform doesn't survive Q3's "GocciaScript will adopt the LWPT-canonical version wholesale" — every marker becomes "GocciaScript will get this when it adopts", so the split has no operational meaning.
- **Collapse to one neutral marker.** Rejected for the same reason: even one marker implies the canonical-source has a reference point worth differentiating from, which is exactly the framing the ADR rejects.
- **Keep existing markers as historical bookmarks, no new ones.** Rejected: half-state where the constraint says "no patch markers" but the codebase still has them; review friction without payoff.

### What about ADR-0003?

- **Supersede via head-of-file amendment box; keep ADR-0003 in the repo.** *Chosen.* New readers find ADR-0017 first via the ADR index; ADR-0003 stays as historical record of the upstream-fork framing that didn't survive contact with reality. The amendment box is the same pattern ADR-0014 used for the workspaces addition.
- **Rewrite ADR-0003 in place.** Rejected: erases the signal that the framing was tried + corrected.
- **Delete ADR-0003.** Rejected: breaks inbound references from other ADRs + commits; loses historical context.
- **Both an amendment to ADR-0003 *and* a new ADR-0017.** Rejected: redundancy without payoff once the amendment box routes forward.

### Hard Constraint replacement in AGENTS.md

The retired constraint: *"Vendored units stay verbatim. No rename, no reformat, no refactor. Patches use `{ [gpm patch] }` / `{ [LWPT patch] }` markers. Permanent fork per ADR-0003."* Every clause is now wrong (vendored → gone; named exceptions ARE canonical; patch markers gone; permanent-fork model superseded).

- **"Packages own their contents."** *Chosen.* Root LWPT discovers `packages/<name>/` via `[workspaces]` and consumes each package's published API; it does not modify a package's source from outside. Each package owns its versioning, format scope, lifecycle hooks, test policy, and public surface. `[format] exclude = ["packages/**"]` in the root `lwpt.toml` is the mechanism; reviewers enforce it for direct edits. Companion line: **"No patch markers."** Git history is the record.
- **Per-package AGENTS.md** (each `packages/<name>/AGENTS.md` declares package-local Hard Constraints; root just points). Rejected as overkill — most packages don't need their own AGENTS.md today.
- **Minimal: just "root formatter does not reach into `packages/`"; no broader package-autonomy claim.** Rejected: the autonomy claim is the actual policy; the formatter exclusion is one mechanism implementing it.

## Consequences

- **The doc set realigns.** `docs/vendored.md` → renamed `docs/packages.md`; whole-file rewrite under the canonical-source framing. `CONTEXT.md`'s `Vendored` + `Graduation` entries are retired / reframed (Package is the umbrella; Graduation is a per-package relocation event, not a state transition of identity). `AGENTS.md` + `CLAUDE.md`'s bootstrap-graduation arc paragraph is rewritten; the "Vendored units stay verbatim" Hard Constraint is replaced with "Packages own their contents" + "No patch markers".
- **the sweeps patch markers from the codebase.** Each `{ [gpm patch] }` / `{ [LWPT patch] }` comment in `packages/httpclient/source/HTTPClient.pas` (four sites), `packages/cli/source/CLI.{Parser,Options,Help}.pas`, `packages/semver/source/Semver.pas`, `source/Platform.pas` is rewritten to a plain Pascal comment that preserves the *why* (e.g. *"Buffer also holds body-prefix bytes that arrive in the same recv as the headers; a Copy(PAnsiChar) would truncate them at the first #0. Hence the byte-safe AppendRawBytes."* in HTTPClient.pas) without the marker prefix.
- **GocciaScript's adoption is a multi-wave effort on the GocciaScript side.** ADR-0017 commits to Path A as the direction; the actual migration (extracting GocciaScript's own non-LWPT shared utilities like `BCP47`, `BOM`, `BigInteger`, `JSONParser`, `HashMap`, `ICU*`, `TextSemantics`, `TimingUtils` into GocciaScript's own `packages/<name>/` tree; rewriting `build.pas` → `gocciascript/lwpt.toml`'s `[build]`; retiring `format.pas`; landing the `lwpt install` precondition) is GocciaScript-side work tracked separately.
- **Bootstrap chicken-and-egg still bites; today's solution survives.** LWPT cannot fetch its own deps without HTTPClient existing in some form. Today this is satisfied by HTTPClient being in `packages/httpclient/source/` (in the monorepo) — the bootstrap (`bootstrap.sh` / `bootstrap.bat` / `scripts/bootstrap.pas`) compiles `source/lwpt.pas` directly with `-Fu packages/httpclient/source` (alongside every other package), so no separate "slim embedded" copy is needed. Post-Phase-2 (when `packages/httpclient/` graduates to a standalone repo), LWPT will need to keep a bootstrap copy in its repo (mechanism TBD: continue maintaining `packages/httpclient/` alongside the standalone, or snapshot the standalone at a fixed commit under `bootstrap/`); the choice gets its own follow-on ADR at graduation time.
- **Graduation criteria stay deliberately fuzzy.** A package leaves LWPT's monorepo for its own repo when warranted by signals like: (a) external (non-`frostney/*`) contributor base develops, (b) package's release cadence diverges meaningfully from LWPT's, (c) package's CI / docs / tests warrant dedicated workflows. Any one signal is enough to consider; none alone forces. Each graduation gets its own ADR documenting the trigger + transition plan.
- **CI guards from [ADR-0016](./0016-tls-backend-per-platform.md) stand unchanged.** The SChannel-only Windows check in `ci.yml` + `release.yml` continues to enforce per-platform TLS-backend correctness; ADR-0017 doesn't touch the TLS story.
- **Per-package versioning is independent semver 2.0.0.** Each `packages/<name>/lwpt.toml` carries its own `[package].version`. Bumps happen per-package, not in lockstep with LWPT's root version. The root `lwpt.toml` doesn't carry an aggregate "all packages at X.Y.Z" version.
- **Cross-package edits go through the package's own contract.** Even from a contributor sitting at the LWPT root: a fix to `packages/httpclient/source/HTTPClient.pas` opens a PR against the package's own contract (its tests, its review process), not against the root LWPT manifest. Today this is procedural (no separate CI / no per-package CODEOWNERS); the constraint sets the cultural expectation for when those mechanisms grow.
- **No drift-check infrastructure.** ADR-0003's "no `vendored.toml`, no `verify-vendor.pas`" rationale survives: with packages being canonical-at-LWPT (not vendored from elsewhere), there's nothing external to drift against. The transition-window divergence with GocciaScript is documented in `docs/packages.md` (the renamed `vendored.md`) as a known state, not a defect to actively monitor.
- **The CONTEXT.md glossary loses two terms and gains zero.** `Vendored` and `Graduation` retire; `Package` already exists as the umbrella term (per Q1 in this ADR's grilling); the consumption-pattern words (`Workspace`, `Monorepo dep`, `External-path dep`) stay as descriptive, not categorical, distinctions.
- **External documentation framing is consumer-agnostic.** README + quick-start + tooling docs talk about "packages live in `packages/<name>/`; install resolves them" without naming GocciaScript by default. ADR-0017 + `docs/packages.md` carry the GocciaScript-specific commitment + transition story.
