### New Features

- One-binary toolkit (`lwpt`) with seven subcommands: `init`, `install`, `build`, `format`, `test`, `repair`, `run`. See [`AGENTS.md`](./AGENTS.md) for the Quick Reference.
- Manifest-driven build pipeline: `lwpt.toml` declares `[package]`, `[dependencies]`, `[build]`, `[format]`, `[workspaces]`, `[sources]`, and the `[preinstall]` / `[postinstall]` / `[prebuild]` / `[postbuild]` / `[pretest]` / `[posttest]` lifecycle hooks. See [`docs/build-system.md`](./docs/build-system.md).
- Workspace-package monorepo support per [ADR-0014](./docs/adr/0014-packages-extraction.md) + [ADR-0017](./docs/adr/0017-packages-lwpt-canonical.md). Five LWPT-canonical packages: `httpclient`, `cli`, `semver`, `toml`, `testing`.
- Source kinds per [ADR-0009](./docs/adr/0009-source-syntax-and-tag-resolution.md): bare `owner/repo` (GitHub), `gitlab:` / `bitbucket:` prefixes, custom `[sources]` prefixes (Gitea / Forgejo / self-hosted), `https://...` arbitrary tarballs, local-path with `local:` prefix. Tag resolution via git smart-HTTP.
- Zero-install by default per [ADR-0002](./docs/adr/0002-lwpt-namespace-zero-install.md): `.lwpt/modules/` + `.lwpt/archives/` are committed; `git clone` works without `lwpt install`.
- Lockfile schema v3 with two-hash `--frozen` verification (archive + extracted-tree SHA-256).
- Cross-process install lock (`O_CREAT|O_EXCL` on Unix; `LockFileEx` lands with the Windows CI work).
- Atomic file writes via `.lwpt/tmp/` with EXDEV-rename fallback.
- Per-platform TLS backend per [ADR-0016](./docs/adr/0016-tls-backend-per-platform.md): SChannel on Windows, SecureTransport on macOS, system OpenSSL on Linux. CI guard prevents OpenSSL DLL dependencies on Windows.
- `lwpt run <name>` subcommand per [ADR-0013](./docs/adr/0013-run-subcommand-and-build-rename.md): invokes user-declared run-scripts or aliases built-in subcommands.
- `lwpt --version` reports the binary's version, derived at compile time from `[package].version` in `lwpt.toml` (no drift possible).
- Pre-1.0 cross-platform CI (Linux x86_64 + aarch64, Windows x86_64, macOS arm64 + Intel) via cached cross-FPC toolchain build. PRs run a fast Ubuntu-only gate (`pr.yml`); push-to-main runs the full matrix (`ci.yml`); tag push runs the release pipeline (`release.yml`).
- Install scripts at `scripts/install.{sh,ps1}` for one-liner install on macOS / Linux / Windows.

### Bug Fixes

- HTTPClient byte-safety: replaced `Copy(PAnsiChar(...))` with a byte-safe `AppendRawBytes` accumulator at four sites on the header-recv + chunked-body paths. Without this, every binary download corrupted at the first `#0` byte.

### Internal

- Migrated from a spike codebase (formerly `lwptk`) to a production toolkit (`lwpt`). Project name + binary + module dir + cfg file all derive from a single `PROGRAM_NAME` constant per [ADR-0001](./docs/adr/0001-program-name-as-constant.md).
- 17 ADRs document the architectural decisions that shaped the v1 design. See [`docs/README.md`](./docs/README.md#decision-records) for the index.
